# Плагин автоответчика QQ

Плагин подключается к QQ через протокол OneBot и обеспечивает интеллектуальные автоответы на основе прав доступа. Поддерживаются личные сообщения и групповые чаты, а также интеграция с ИИ-диалогами.

## Запуск и онбординг

1. Скачайте реализацию OneBot (рекомендуется NapCat). В примерах ниже используется именно NapCat.
   Откройте:
   ```text 
   https://github.com/NapNeko/NapCatQQ/releases
   ```
   Выберите любой релиз для загрузки (рекомендуется `NapCat.Shell.zip`).
2. Запустите NapCat и откройте страницу настройки NapCat.
3. На левой панели выберите **Сетевые настройки**.
4. Добавьте **WS Server**, включите его и сохраните.
5. Запустите плагин.
6. Вернитесь назад один раз.
7. Снова откройте панель этого плагина.
8. Пройдите шаги руководства по порядку.

Текущие особенности потока:
- На этапе `startup` плагин автоматически создаёт бизнес-конфиг, если его ещё нет.
- NapCat сначала запускается из заданного каталога выполнения, а если он не указан — из каталога по умолчанию.
- После генерации QR-кода для входа NapCat копирует его в статический каталог плагина; в интерфейсе можно нажать **Обновить QR-код**, чтобы повторно синхронизировать и показать актуальное изображение.
- Во фронтенд-интерфейсе можно напрямую управлять:
  - адресом OneBot / Token / PATH
  - запуском NapCat на переднем плане или в фоне
  - доверенными пользователями / доверенными группами
  - включением и остановкой автоответчика

## Возможности

- Поддержка протокола OneBot через NapCat
- Многоуровневые пользовательские права: `admin`, `trusted`, `normal`
- Управление правами групп: `trusted`, `open`, `normal`
- Интеграция AI-диалогов через OmniOfflineClient
- Синхронизация памяти для личных диалогов администратора
- Функция пересылки обычных сообщений администратору с вероятностью
- Режим open-групп для ответов без `@`
- Управление пользовательскими прозвищами
- Автоматическое переподключение WebSocket с экспоненциальной задержкой

## Настройка

Рекомендуется настраивать всё через интерфейс плагина в разделе **Настройка сервиса QQ OneBot**. При первом запуске бизнес-конфиг создаётся автоматически. Основные поля:

- `onebot_url`: WebSocket-адрес сервиса OneBot, по умолчанию `ws://127.0.0.1:3001`
- `token`: токен доступа OneBot
- `napcat_directory`: каталог выполнения NapCat
- `show_napcat_window`: `true` — запуск на переднем плане с видимой консолью, `false` — фоновый запуск
- `trusted_users`: список доверенных пользователей
- `trusted_groups`: список доверенных групп
- `normal_relay_probability`: вероятность пересылки обычных сообщений хозяину
- `truth_reply_probability`: вероятность инициативного ответа в open-группах

### Таблица параметров

| Ключ | Тип | Описание |
|------|-----|----------|
| `onebot_url` | string | WebSocket-адрес сервиса OneBot |
| `token` | string | Токен доступа к сервису OneBot (если требуется) |
| `trusted_users` | array | Список доверенных пользователей с QQ-номером, уровнем прав и ником |
| `trusted_groups` | array | Список доверенных групп с номером группы и уровнем прав |
| `normal_relay_probability` | float | Вероятность пересылки обычных личных/групповых сообщений хозяину |
| `truth_reply_probability` | float | Вероятность прямого ответа в `open`-группах без `@` |

## Уровни прав

### Пользовательские права

| Уровень | Значение | Поведение |
|---------|----------|-----------|
| `admin` | Администратор | Прямой ответ в личке, синхронизация диалога в память, обращение как к «хозяину» |
| `trusted` | Доверенный пользователь | Прямой ответ в личке, без синхронизации памяти, можно задать ник |
| `normal` | Обычный пользователь | Без прямого ответа, сообщение может быть переслано администратору |
| `none` | Неавторизованный | Сообщение игнорируется |

### Права групп

| Уровень | Значение | Поведение |
|---------|----------|-----------|
| `trusted` | Доверенная группа | Ответ только если бота упомянули через `@` |
| `open` | Открытая группа | Может отвечать без `@`, использует временный контекст и карточку персонажа, но не пишет в память |
| `normal` | Обычная группа | Не отвечает напрямую, может переслать администратору |
| `none` | Неавторизованная | Сообщение игнорируется |

## Дополнительно

### 1. Активная отправка сообщений

Через отдельные entry в панели можно сначала попросить ИИ сгенерировать сообщение в нужном стиле, а потом активно отправить его выбранному пользователю или группе.

Примечания:
- `message` теперь означает промпт для ИИ, а не финальный текст, который будет отправлен как есть.
- Используются текущие настройки персонажа и модели.
- При активной отправке в личку может быть прочитан контекст памяти, но этот отправленный текст сам в память не записывается.
- При активной отправке в группу память не обновляется.
- Личная entry использует `target`: это может быть числовой QQ ID или ник, уже добавленный в список доверенных пользователей.
- `group_id` должен быть строкой, состоящей только из цифр.
- `message` не может быть пустым.
- Автоответчик должен быть уже запущен, а OneBot должен быть подключён, иначе entry завершится ошибкой.

### 2. Остановка плагина

При остановке выполняется:
- разрыв WebSocket-соединения
- очистка рабочих ресурсов

## Частые вопросы

### 1. Не удаётся подключиться к OneBot

**Проблема**: в логах видно `Failed to connect to OneBot`

**Что проверить**:
- работает ли NapCat корректно (порт 3001)
- правильно ли указан `onebot_url`
- корректен ли `token`

### 2. Бот не отвечает

**Проблема**: сообщение отправлено, но ответа нет

**Что проверить**:
- есть ли отправитель в `trusted_users`
- верный ли уровень прав (`normal` пользователям напрямую не отвечаем)
- доходят ли сообщения до плагина по логам
- в `trusted`-группах бот должен быть упомянут через `@`
- в `open`-группах `@` не требуется, если всё настроено правильно

### 3. Ошибка синхронизации памяти

**Проблема**: в логах появляется ошибка синхронизации памяти

**Что проверить**:
- запущен ли Memory Server
- в память синхронизируются только личные диалоги администратора
- групповые диалоги (`open` в том числе) используют только временный контекст и не записываются в хранилище памяти

### 4. Не работает пересылка обычных сообщений

**Проблема**: сообщения обычных пользователей не пересылаются администратору

**Что проверить**:
- задан ли администратор (`level = "admin"`)
- корректно ли настроен `normal_relay_probability` (по умолчанию 0.1 = 10%)
- срабатывала ли пересылка по логам

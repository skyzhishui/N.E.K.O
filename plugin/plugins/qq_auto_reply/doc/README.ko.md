# QQ 자동 응답 플러그인

OneBot 프로토콜을 통해 QQ에 연결하고, 권한에 따라 지능형 자동 응답을 제공하는 플러그인입니다. 개인 대화와 그룹 메시지를 모두 지원하며 AI 대화 기능도 통합되어 있습니다.

## 시작 및 안내

패널을 연 뒤 다음 순서대로 진행하세요.

1. OneBot 구현체를 준비합니다(NapCat 권장). 아래 예시는 NapCat 기준입니다.
   열기:
   ```text 
   https://github.com/NapNeko/NapCatQQ/releases
   ```
   아무 배포판이나 선택할 수 있지만 `NapCat.Shell.zip` 을 추천합니다.
2. NapCat을 실행하고 NapCat 설정 페이지를 엽니다.
3. 왼쪽 도구 모음에서 **네트워크 설정** 을 선택합니다.
4. **WS 서버** 를 추가하고 활성화한 뒤 저장합니다.
5. 플러그인을 시작합니다.
6. 한 번 뒤로 돌아갑니다.
7. 이 플러그인 패널을 다시 엽니다.
8. 안내 순서대로 설정합니다.

현재 흐름 설명:
- 플러그인의 `startup` 단계에서 비즈니스 설정 파일이 없으면 자동으로 생성됩니다.
- NapCat은 먼저 설정된 실행 디렉터리에서 시작하며, 비어 있으면 기본 디렉터리로 돌아갑니다.
- NapCat이 로그인 QR 코드를 생성하면 플러그인 정적 디렉터리로 복사됩니다. UI의 **QR 코드 새로고침** 을 누르면 최신 이미지로 다시 동기화해 표시합니다.
- 프런트엔드 UI에서 직접 관리할 수 있는 항목:
  - OneBot 주소 / Token / PATH
  - NapCat 전경/백그라운드 실행
  - 신뢰 사용자 / 신뢰 그룹
  - 자동 응답 시작 / 중지

## 기능 특징

- NapCat 기반 OneBot 프로토콜 지원
- `admin` / `trusted` / `normal` 3단계 사용자 권한
- `trusted` / `open` / `normal` 3단계 그룹 권한
- OmniOfflineClient 기반 AI 답변 생성
- 관리자 개인 대화를 Memory Server와 동기화
- 일반 사용자 메시지에 대한 확률 기반 전달 기능
- `@` 없이도 응답 가능한 open 그룹 모드
- 신뢰 사용자용 별명 관리
- WebSocket 끊김 후 지수 백오프로 자동 재연결

## 설정 안내

기본적으로 플러그인 UI의 **QQ OneBot 서비스 설정** 에서 설정하는 것을 권장합니다. 최초 시작 시 비즈니스 설정 파일은 자동 생성됩니다. 주요 항목은 다음과 같습니다.

- `onebot_url`: OneBot WebSocket 주소, 기본값 `ws://127.0.0.1:3001`
- `token`: OneBot 접근 토큰
- `napcat_directory`: NapCat 실행 디렉터리
- `show_napcat_window`: `true`면 전경 실행(콘솔 표시), `false`면 백그라운드 실행
- `trusted_users`: 신뢰 사용자 목록
- `trusted_groups`: 신뢰 그룹 목록
- `normal_relay_probability`: 일반 메시지를 주인에게 전달할 확률
- `truth_reply_probability`: open 그룹에서 적극적으로 답장할 확률

### 설정 항목 표

| 키 | 타입 | 설명 |
|----|------|------|
| `onebot_url` | string | OneBot 서비스의 WebSocket 주소 |
| `token` | string | OneBot 서비스 접근 토큰(필요한 경우) |
| `trusted_users` | array | QQ 번호, 권한 레벨, 별명을 포함한 신뢰 사용자 목록 |
| `trusted_groups` | array | 그룹 번호와 권한 레벨을 포함한 신뢰 그룹 목록 |
| `normal_relay_probability` | float | 일반 사용자/일반 그룹 메시지를 주인에게 전달할 확률 |
| `truth_reply_probability` | float | `open` 그룹에서 `@` 없이 직접 답변할 확률 |

## 권한 레벨

### 사용자 권한

| 레벨 | 의미 | 동작 |
|------|------|------|
| `admin` | 관리자 | 개인 채팅에 직접 답장, 대화를 메모리로 동기화, "주인"으로 호칭 |
| `trusted` | 신뢰 사용자 | 개인 채팅에 직접 답장, 메모리 동기화 없음, 별명 설정 가능 |
| `normal` | 일반 사용자 | 직접 답장하지 않음, 확률적으로 관리자에게 전달 |
| `none` | 미허용 | 메시지 무시 |

### 그룹 권한

| 레벨 | 의미 | 동작 |
|------|------|------|
| `trusted` | 신뢰 그룹 | 봇이 `@` 되었을 때만 AI 답변 |
| `open` | 개방 그룹 | `@` 없이도 직접 답변 가능. 임시 세션 메모리와 캐릭터 카드 문맥을 사용하며 메모리 저장소에는 기록하지 않음 |
| `normal` | 일반 그룹 | 직접 답장하지 않고 확률적으로 관리자에게 전달 |
| `none` | 미허용 | 메시지 무시 |

## 추가 메모

### 1. 능동 발신

전용 엔트리를 호출하면 먼저 AI가 캐릭터다운 문장을 생성한 뒤, 지정한 사용자나 그룹으로 능동 발신할 수 있습니다.

설명:
- `message` 는 최종 전송 문장이 아니라 AI에게 주는 지시 내용입니다.
- 기존 캐릭터 설정과 모델 설정을 그대로 사용합니다.
- 개인 채팅 능동 발신은 메모리 문맥을 읽어 생성할 수 있지만, 이 발신 자체는 메모리에 다시 기록되지 않습니다.
- 그룹 능동 발신도 메모리에 기록되지 않습니다.
- 개인 발신 entry 는 `target` 을 사용하며, 숫자 QQ ID 또는 신뢰 사용자 목록에 등록된 별명을 지정할 수 있습니다.
- `group_id` 는 숫자로만 이루어진 문자열이어야 합니다.
- `message` 는 비워둘 수 없습니다.
- 먼저 자동 응답이 시작되어 있고 OneBot이 연결되어 있어야 합니다.

## 자주 묻는 질문

### 1. OneBot에 연결할 수 없음

**문제**: 로그에 `Failed to connect to OneBot` 가 표시됨

**해결 방법**:
- NapCat이 정상적으로 실행 중인지(포트 3001) 확인
- `onebot_url` 값이 올바른지 확인
- `token` 값이 맞는지 확인

### 2. 봇이 답장하지 않음

**문제**: 메시지를 보냈는데 답장이 없음

**해결 방법**:
- 발신자가 `trusted_users` 에 포함되어 있는지 확인
- 권한 레벨 확인 (`normal` 사용자는 직접 답장을 받지 않음)
- 로그에서 메시지가 수신되었는지 확인
- `trusted` 그룹에서는 봇을 `@` 했는지 확인
- `open` 그룹은 설정이 올바르면 `@` 없이도 답장 가능

### 3. 메모리 동기화 실패

**문제**: 로그에 메모리 동기화 실패가 표시됨

**해결 방법**:
- Memory Server가 실행 중인지 확인
- 메모리로 동기화되는 것은 관리자와의 개인 채팅뿐임
- 그룹 대화(`open` 포함)는 임시 문맥만 유지하며 메모리 저장소에는 기록하지 않음

### 4. 전달 기능이 동작하지 않음

**문제**: 일반 사용자 메시지가 관리자에게 전달되지 않음

**해결 방법**:
- 관리자(`level = "admin"`)가 설정되어 있는지 확인
- `normal_relay_probability` 값 확인(기본값 0.1 = 10%)
- 로그에서 전달이 실제로 발생했는지 확인

# QQ 自動返信プラグイン

OneBot プロトコル経由で QQ に接続し、権限に応じたインテリジェントな自動返信を行うプラグインです。個人チャットとグループメッセージの両方に対応し、AI 会話機能も統合されています。

## 起動とガイド

1. OneBot 実装を用意してください（NapCat 推奨）。以下では NapCat を例にします。
   開く:
   ```text 
   https://github.com/NapNeko/NapCatQQ/releases
   ```
   任意の配布物を選んでください（推奨は `NapCat.Shell.zip`）。
2. NapCat を起動し、NapCat の設定ページを開きます。
3. 左側ツールバーから **ネットワーク設定** を選びます。
4. **WS サーバー** を追加して有効化し、その後保存します。
5. プラグインを起動します。
6. 一度戻ります。
7. もう一度このプラグインのパネルを開きます。
8. ガイドの順番通りに設定します。

現在の流れの補足:
- プラグインの `startup` で、ビジネス設定ファイルがまだ無ければ自動作成されます。
- NapCat はまず設定済みの実行ディレクトリから起動し、未設定なら既定ディレクトリへフォールバックします。
- NapCat がログイン用 QR コードを生成すると、それをプラグインの静的ディレクトリへコピーします。UI の **QR コードを更新** を押すと最新画像へ同期して表示します。
- フロントエンド UI では次を直接管理できます。
  - OneBot アドレス / Token / PATH
  - NapCat の前面 / 背景起動
  - 信頼ユーザー / 信頼グループ
  - 自動返信の開始 / 停止

## 機能概要

- NapCat を利用した OneBot プロトコル対応
- `admin` / `trusted` / `normal` の 3 段階ユーザー権限
- `trusted` / `open` / `normal` の 3 段階グループ権限
- OmniOfflineClient を使った AI 返信生成
- 管理者との個人チャットを Memory Server と同期
- 通常ユーザー向けの確率転送モード
- `@` なしでも応答できる open グループモード
- 信頼ユーザー向けニックネーム管理
- WebSocket 切断後の指数バックオフ再接続

## 設定について

基本的にはプラグイン UI の **QQ OneBot サービス設定** から設定するのをおすすめします。初回起動時にビジネス設定ファイルは自動生成されます。主な項目は以下の通りです。

- `onebot_url`: OneBot の WebSocket アドレス。既定値は `ws://127.0.0.1:3001`
- `token`: OneBot アクセストークン
- `napcat_directory`: NapCat 実行ディレクトリ
- `show_napcat_window`: `true` なら前面起動でコンソール表示、`false` なら背景起動
- `trusted_users`: 信頼ユーザー一覧
- `trusted_groups`: 信頼グループ一覧
- `normal_relay_probability`: 通常メッセージを主人へ転送する確率
- `truth_reply_probability`: open グループで主动返信する確率

### 設定項目一覧

| キー | 型 | 説明 |
|-----|----|------|
| `onebot_url` | string | OneBot サービスの WebSocket アドレス |
| `token` | string | OneBot サービスのアクセストークン（必要な場合） |
| `trusted_users` | array | QQ 番号・権限レベル・ニックネームを含む信頼ユーザー一覧 |
| `trusted_groups` | array | グループ番号・権限レベルを含む信頼グループ一覧 |
| `normal_relay_probability` | float | 通常ユーザー / 通常グループのメッセージを主人へ転送する確率 |
| `truth_reply_probability` | float | `open` グループで `@` なし返信を発動する確率 |

## 権限レベル

### ユーザー権限

| レベル | 意味 | 挙動 |
|-------|------|------|
| `admin` | 管理者 | 個人チャットへ直接返信、会話は記憶へ同期、「主人」と呼ばれる |
| `trusted` | 信頼ユーザー | 個人チャットへ直接返信、記憶同期なし、ニックネーム設定可能 |
| `normal` | 通常ユーザー | 直接返信しない。確率で管理者へ転送 |
| `none` | 未許可 | メッセージを無視 |

### グループ権限

| レベル | 意味 | 挙動 |
|-------|------|------|
| `trusted` | 信頼グループ | Bot が `@` された場合のみ AI 返信 |
| `open` | 開放グループ | `@` なしでも直接返信可。臨時セッション記憶とキャラカード文脈を利用。記憶ストレージには書き込まない |
| `normal` | 通常グループ | 直接返信せず、確率で管理者へ転送 |
| `none` | 未許可 | メッセージを無視 |

## 追加メモ

### 1. 能動送信

専用エントリを呼び出すことで、Bot にまず AI らしい文面を生成させ、その後で対象ユーザーやグループへ能動送信できます。

補足:
- `message` は最終送信テキストではなく、AI への指示文です。
- 既存のキャラ設定とモデル設定をそのまま使います。
- 個人チャットへの能動送信では記憶コンテキストを参照する場合がありますが、この送信自体は記憶へ書き戻しません。
- グループへの能動送信も記憶へは書き込みません。
- 個人向け entry は `target` を使います。数値の QQ ID、または信頼ユーザー一覧に設定済みのニックネームを指定できます。
- `group_id` は数字だけの文字列である必要があります。
- `message` は空にできません。
- あらかじめ自動返信が起動していて、OneBot が接続済みである必要があります。

### 2. プラグイン停止

停止時には以下が実行されます。
- WebSocket 切断
- 実行中リソースのクリーンアップ

## FAQ

### 1. OneBot に接続できない

**症状**: ログに `Failed to connect to OneBot` が出る

**対処**:
- NapCat が正しく起動しているか（ポート 3001）確認
- `onebot_url` が正しいか確認
- `token` が正しいか確認

### 2. Bot が返信しない

**症状**: メッセージを送っても返信がない

**対処**:
- 送信者が `trusted_users` に入っているか確認
- 権限レベルを確認（`normal` ユーザーには直接返信しません）
- ログで受信できているか確認
- `trusted` グループでは Bot を `@` したか確認
- `open` グループでは `@` は不要です（設定が正しければ）

### 3. 記憶同期に失敗する

**症状**: ログに記憶同期失敗が出る

**対処**:
- Memory Server が起動しているか確認
- 記憶へ同期されるのは管理者との個人チャットだけです
- グループ（`open` 含む）は臨時コンテキストのみで、記憶ストレージへは書き込みません

### 4. 転送機能が動かない

**症状**: 通常ユーザーのメッセージが管理者へ転送されない

**対処**:
- 管理者（`level = "admin"`）が設定されているか確認
- `normal_relay_probability` を確認（既定値 0.1 = 10%）
- ログ上で転送が発火しているか確認

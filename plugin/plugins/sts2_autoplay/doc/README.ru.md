# Быстрый старт

`sts2_autoplay` используется для подключения в N.E.K.O локального состояния *Slay the Spire 2*, которое публикует `STS2 AI Agent`. Плагин может читать текущую ситуацию, выполнять допустимые действия, автоматически играть по стратегии, позволять кошкодевочке выбирать одну карту, отправлять информацию наблюдения на фронтенд, а также позволять кошкодевочке отправлять мягкие подсказки в фоновой задаче, чтобы влиять на следующий раунд принятия решений.

## Руководство по использованию

### Получение MOD

Используя Git:
```text
https://github.com/CharTyr/STS2-Agent/releases
```

### Установка игрового Mod

В Steam нажмите правой кнопкой мыши по *Slay the Spire 2* и выберите Управление -> Просмотреть локальные файлы.

Стандартный каталог игры Steam обычно выглядит примерно так:

```text
...\Steam\steamapps\common\Slay the Spire 2
```

Скопируйте mod `STS2 AI Agent` в каталог `mods/` внутри папки игры.

Если в каталоге *Slay the Spire 2* нет папки `mods`, создайте её вручную.

```text
Использование модов может привести к потере сохранений. Сделайте резервную копию или компенсируйте это через консоль (в главном меню Slay the Spire нажмите клавишу "~", введите "unlock all", и все персонажи и уровни сложности будут разблокированы).
```

После установки структура каталога должна быть примерно такой:

```text
Slay the Spire 2/
  mods/
    STS2AIAgent.dll
    STS2AIAgent.pck
    mod_id.json
```

### Запуск игры и проверка интерфейса

Сначала запустите игру обычным способом, чтобы Mod загрузился вместе с ней.

При первом переключении в режим с модом игра может один раз вылететь. Это нормально — просто запустите игру снова.

После загрузки мода в N.E.K.O включите Cat Paw, активируйте плагин, откройте панель плагина и вручную запустите плагин Slay the Spire.

### Доступные команды

【Сыграй карту】【Автоиграй за меня】【Пройди этаж】【Как я сыграл】【Стоп】
【Сыграй одну карту】【Сыграй определённую карту】【Порекомендуй карту】…… и подобные фразы.

## Обзор возможностей

- Подключается к локальному HTTP-сервису `STS2 AI Agent` и считывает состояние игры.
- Поддерживает ручное выполнение одного шага, фоновую полуавтоматическую игру, паузу, возобновление и остановку.
- Поддерживает три режима принятия решений: `full-program`, `half-program` и `full-model`.
- Поддерживает загрузку стратегических документов по персонажу; файлы стратегий находятся в `strategies/`.
- Поддерживает выбор одной карты кошкодевочкой: выбирается только одна карта из текущих доступных действий `play_card`, сначала отправляется причина, затем выполняется действие.
- Поддерживает мягкие подсказки кошкодевочки: пользователь или сама кошкодевочка могут отправить инструкцию на естественном языке, и следующий раунд решения LLM будет её учитывать.
- Поддерживает фоновые отчёты наблюдения: отправляет во фронтенд текущий этаж, бой, руку, намерения врагов, обоснование LLM и т. д.
- Поддерживает защитные механизмы: пауза при низком HP, замедление в Boss-боях/опасных атаках, автоматическое возобновление после восстановления HP, стратегию выживания на грани смерти, максимизацию выгоды и оценку синергии.

## Конфигурация плагина

Файл конфигурации: `plugin.toml`

### Базовая конфигурация

| Параметр | Значение по умолчанию | Описание |
| --- | --- | --- |
| `base_url` | `http://127.0.0.1:8080` | Адрес локального Agent для Spire. |
| `connect_timeout_seconds` | `5` | Таймаут подключения в секундах. |
| `request_timeout_seconds` | `15` | Таймаут запроса в секундах. |
| `poll_interval_idle_seconds` | `3` | Интервал опроса в состоянии бездействия. |
| `poll_interval_active_seconds` | `1` | Интервал опроса во время работы автоигры. |
| `action_interval_seconds` | `1.5` | Дополнительный интервал между действиями. |
| `post_action_delay_seconds` | `0.5` | Задержка после выполнения действия, чтобы подождать стабилизации ситуации. |
| `autoplay_on_start` | `false` | Должна ли игра начинаться автоматически после запуска плагина. |
| `semi_auto_autoplay` | `true` | Создавать ли контекст полуавтоматической задачи при запуске автоигры. |
| `mode` | `half-program` | Текущий режим автоигры. |
| `character_strategy` | `defect` | Имя стратегии персонажа, соответствующее `strategies/<name>.md`. |
| `max_consecutive_errors` | `3` | Максимальное число последовательных ошибок; после превышения считается, что соединение потеряно. |
| `push_notifications` | `true` | Исторически сохранённое поле. |
| `event_stream_enabled` | `false` | Зарезервированное поле, на данный момент фактически не используется. |

### Режимы принятия решений

`mode` поддерживает следующие значения, а также их китайские псевдонимы:

| Режим | Китайский псевдоним | Описание |
| --- | --- | --- |
| `full-program` | `全程序` | Чисто программная эвристика, без вызова модели. |
| `half-program` | `半程序` | Сначала выполняются программные предварительные проверки, затем один вызов модели для принятия решения, после чего выполняется проверка допустимости/откат. |
| `full-model` | `全模型` | Два вызова модели: сначала reasoning, затем final action; между ними выполняются программные проверки, а в конце — повторная проверка допустимости. |

### Стратегия персонажа

`character_strategy` ищет документ стратегии по пути `strategies/<name>.md`. Встроенные стратегии на данный момент:

- `defect`
- `ironclad`
- `silent_hunter`
- `necrobinder`
- `regent`

Вы можете добавить новые Markdown-файлы в `strategies/`, чтобы расширить список стратегий. Например:

```text
strategies/my_strategy.md
```

После этого установите значение конфигурации или параметра входа как:

```text
my_strategy
```

### Отправка во фронтенд и наблюдение кошкодевочки

| Параметр | Значение по умолчанию | Описание |
| --- | --- | --- |
| `llm_frontend_output_enabled` | `true` | Нужно ли активно отправлять действия/ошибки автоигры во фронтенд. |
| `llm_frontend_output_probability` | `0.15` | Вероятность отправки обычных действий; ограничивается диапазоном `0.0 ~ 1.0`. Ошибки отправляются всегда. |
| `neko_reporting_enabled` | `true` | Отправлять ли отчёты наблюдения кошкодевочки. |
| `neko_report_interval_steps` | `1` | Через сколько шагов автоигры отправлять один отчёт наблюдения; минимум `1`. |
| `neko_commentary_enabled` | `true` | Генерировать ли живые комментарии кошкодевочки внутри отчёта наблюдения. Если отключено, структурированный отчёт всё равно будет отправляться, но `live_commentary.text` останется пустым. |
| `neko_commentary_probability` | `0.65` | Вероятность срабатывания обычных комментариев низкого приоритета; ограничивается диапазоном `0.0 ~ 1.0`. Сцены высокого приоритета, такие как низкое HP, летал или высокий урон, могут обходить эту вероятность. |
| `neko_commentary_min_interval_seconds` | `4` | Минимальный интервал в секундах перед повтором комментария для одной и той же низкоприоритетной сцены, чтобы уменьшить спам и повторяющиеся реплики. |
| `neko_critical_commentary_always` | `true` | Нужно ли всегда озвучивать комментарии срочности `critical` / `high`, например при критическом HP, летале или высоком уроне врага. |
| `neko_guidance_max_queue` | `50` | Максимальная длина очереди мягких подсказок кошкодевочки. |

Отчёты наблюдения кошкодевочки будут содержать упрощённые метаданные `report`, `neko_context`, `live_commentary`, `task` и т. д., чтобы фронтенд или диалоговая логика могли понять, что это «наблюдение процесса», а не уведомление о завершении задачи. Для экономии пользовательских токенов в отправляемом содержимом остаются только текущее действие, HP, рука, враги, тактическое резюме, уже потреблённые подсказки и краткое описание задачи.

`live_commentary` предоставляет фронтенду/TTS короткие поля озвучки: `text`, `scene`, `mood`, `urgency`, `priority`, `tts`, `interrupt`, `tone` и `character_strategy`. Комментарии случайным образом выбираются из набора шаблонов по сценам, чтобы уменьшить повторяемость; также они подстраиваются под стратегию персонажа, например `defect` более рационален, а `ironclad` — более устойчив. Сейчас покрываются критическое HP, низкое HP, летал, входящие атаки врагов, защита, обычный бой, награды, магазин, место отдыха, события, карта, а также комментарии событийного уровня — окончание боя, ключевые реликвии, завершение выбора маршрута и т. п.

### Защитные механизмы и автономные действия

| Параметр | Значение по умолчанию | Описание |
| --- | --- | --- |
| `neko_auto_low_hp_threshold` | `0.3` | Если текущая доля HP падает ниже этого значения, автоигра в фоне самостоятельно ставится на паузу. |
| `neko_auto_safe_hp_threshold` | `0.5` | Когда HP восстанавливается до этого значения, возможно автоматическое возобновление. |
| `neko_auto_dangerous_attack_threshold` | `20` | Автоматически замедляться, если входящий урон врага достигает этого значения и пробивает защиту. |
| `neko_auto_resume_after_low_hp` | `true` | Разрешать ли автоматическое возобновление после восстановления HP после паузы на низком HP. |
| `neko_desperate_enabled` | `true` | Включать ли стратегию выживания на грани смерти. |
| `neko_desperate_hp_threshold` | `0.2` | Соотношение HP, запускающее стратегию выживания на грани смерти. |
| `neko_maximize_enabled` | `true` | Включать ли выбор карт с максимизацией выгоды. |
| `neko_synergy_enabled` | `true` | Включать ли оценку синергии/комбо. |

Текущие автономные действия включают:

- `pause`: пауза при низком HP, ожидание указаний пользователя или кошкодевочки.
- `slow_down`: временное замедление интервала действий во время Boss-боёв или опасных атак.
- `resume`: возобновление после выполнения условия безопасного HP.

## Рекомендуемые формулировки для обычных пользователей

Обычным пользователям не нужно запоминать низкоуровневые входы ниже. Предпочтительно передавать оригинальные слова пользователя в `sts2_neko_command`, и плагин внутренне определит, проверять ли статус, давать ли совет, реально играть карту, выполнить шаг, начать автоигру, поставить на паузу, возобновить, остановить, разобрать недавнюю игру, ответить на вопросы об автоигре или использовать формулировку как мягкую подсказку во время автоигры.

Рекомендуемые правила взаимодействия:

| Формулировка пользователя | Поведение плагина |
| --- | --- |
| `Spire подключен?` / `какая сейчас ситуация` | Проверять только подключение, статус или снимок; не управлять игрой. |
| `как ходить в этом ходу` / `какую карту лучше сыграть` | Рекомендовать только одну карту для игры с объяснением; не играть автоматически. |
| `сыграй за меня карту` / `выбери карту и сыграй` | После явного разрешения выбрать только одну карту из действий `play_card` и сыграть её. |
| `сделай за меня шаг` / `выполни шаг` | После явного разрешения выполнить одно допустимое действие, которое может включать завершение хода, выбор награды или ход по карте. |
| `пройди за меня этаж` / `автоиграй немного` | Запустить полуавтоматическую игру, по умолчанию условие остановки — завершение текущего этажа. |
| `сначала защищайся` / `не жадничай по урону` | Во время автоигры это становится мягкой подсказкой для следующего раунда решений; если автоигра не запущена, консервативно запросить уточнение, не действовать. |
| `как я только что сыграл` / `разбери последнюю карту` | Дать оценку по последнему лёгкому снимку; не управлять игрой. |
| `почему так играешь` / `что ты делаешь` | Во время автоигры отвечать о текущей стратегии и обосновании ситуации; не выполнять дополнительных действий. |
| `пауза` / `продолжай` / `останови` | Соответственно поставить автоигру на паузу, возобновить или остановить. |

Безопасные значения по умолчанию: консультация не управляет игрой, размытые формулировки не выполняют опасные действия; реальные действия выполняются только когда пользователь явно говорит «сыграй за меня», «выполни», «автоиграй», «возьми на себя».

## Точки входа плагина

Следующие точки входа открыты для хоста и могут вызываться напрямую в N.E.K.O. Для сценариев обычных пользователей предпочтительно вызывать `sts2_neko_command`; другие точки входа в основном служат интерфейсами точного контроля для разработчиков.

### `sts2_neko_command`

Главная естественно-языковая точка входа для Slay the Spire. Если пользователь явно не указал низкоуровневый инструмент, предпочтительно вызывать её.

Параметры:

- `command`: обязательный, оригинальные слова пользователя. Например: `как ходить в этом ходу`, `сыграй за меня карту`, `сначала защищайся`, `пауза`.
- `scope`: необязательный, по умолчанию `auto`. Возможные значения: `auto`, `status`, `advice`, `one_card`, `one_action`, `autoplay`, `control`, `guidance`, `review`, `question`, `chat`.
- `confirm`: необязательный, по умолчанию `false`. Используется для подтверждения высокорисковых операций, таких как продолжительная передача управления.

Возврат содержит `intent`, `action`, `executed`, `needs_confirmation`, `summary` и нижестоящий `result`.

### `sts2_health_check`

Проверка доступности локального сервиса Spire Agent.

### `sts2_refresh_state`

Принудительное однократное обновление текущего состояния Spire.

### `sts2_get_status`

Получение информации о состоянии подключения, статусе автоигры, текущем режиме, стратегии персонажа, полуавтоматической задаче, последних ошибках, последних действиях и т. д.

### `sts2_get_snapshot`

Получение последнего кешированного снимка игры и текущих доступных действий.

### `sts2_step_once`

Выполнение одного шага по текущей стратегии.

### `sts2_play_one_card_by_neko`

Позволяет кошкодевочке выбрать и сыграть карту.

Параметры:

- `objective`: необязательный, цель авторизации пользователя. Например: `выбери карту и сыграй за меня`.

Поведение:

1. Считывает текущего игрока, руку, врагов и допустимые действия.
2. Сохраняет только действия `play_card`.
3. Позволяет текущему режиму/стратегии выбрать карту.
4. Сначала отправляет во фронтенд «какую карту собирается сыграть и почему».
5. Повторно проверяет, что действие всё ещё допустимо.
6. Играет карту и отправляет наблюдение о завершении.

Если в данный момент нет карт для игры, возвращает `idle` и отправляет причину неудачи.

### `sts2_start_autoplay`

Запускает фоновый цикл полуавтоматической игры.

Параметры:

- `objective`: необязательный, цель авторизации пользователя. Например: `пройди за меня этаж`.
- `stop_condition`: условие остановки, по умолчанию `current_floor`.

`stop_condition` поддерживает:

- `current_floor`: завершается после прохождения текущего этажа или перехода на следующий.
- `current_combat` / `combat`: завершается после того, как во время задачи был вход в бой и затем выход из него.
- `manual` / `none`: не завершается автоматически, требует ручной остановки.

После запуска плагин создаёт контекст полуавтоматической задачи и отправляет во фронтенд событие старта задачи. По завершении задачи отправляется `semi_auto_task_completed`.

### `sts2_pause_autoplay`

Ставит автоигру на паузу.

### `sts2_resume_autoplay`

Возобновляет автоигру, поставленную на паузу, фоновая задача которой ещё существует. Если фоновой задачи больше нет, безопасно возвращает `idle` и не перезапускает автоигру неявно.

### `sts2_stop_autoplay`

Останавливает автоигру и очищает контекст полуавтоматической задачи.

### `sts2_get_history`

Получение последней истории действий и состояний.

Параметры:

- `limit`: количество возвращаемых записей, по умолчанию `20`, диапазон ограничивается `1 ~ 100`.

### `sts2_send_neko_guidance`

Отправляет фоновой автоигре мягкую подсказку кошкодевочки. Подсказка попадает в очередь и инжектируется в контекст в следующем раунде LLM-решений.

Параметры:

- `content`: обязательный, содержимое подсказки на естественном языке. Например: `сначала защищайся, не торопись с уроном`.
- `step`: необязательный, соответствующий номер шага.
- `type`: необязательный, по умолчанию `soft_guidance`.

### `sts2_set_mode`

Устанавливает режим автоигры.

Параметры:

- `mode`: поддерживает `full-program` / `全程序`, `half-program` / `半程序`, `full-model` / `全模型`.

### `sts2_set_character_strategy`

Устанавливает имя стратегии персонажа.

Параметры:

- `character_strategy`: после нормализации имени сопоставляется с `strategies/<name>.md`. Например, `defect` сопоставляется с `strategies/defect.md`.

### `sts2_set_speed`

Устанавливает параметры скорости и записывает их обратно в локальный `plugin.toml`.

Параметры:

- `action_interval_seconds`
- `post_action_delay_seconds`
- `poll_interval_active_seconds`

## Типичные сценарии использования

### Проверка подключения

1. Запустите *Slay the Spire 2*.
2. Убедитесь, что `http://127.0.0.1:8080/health` доступен.
3. В N.E.K.O вызовите `sts2_health_check`.

### Ручное выполнение одного шага

Вызов:

```text
sts2_step_once
```

Плагин выберет и выполнит одно допустимое действие на основе текущих `mode` и `character_strategy`.

### Дать кошкодевочке сыграть карту

Пользователь может сказать кошкодевочке что-то вроде:

```text
выбери карту и сыграй за меня
```

Хост должен вызвать:

```text
sts2_play_one_card_by_neko
```

Плагин выбирает только из доступных в данный момент карт и не выбирает завершение хода, карту, награду или другие действия.

### Дать кошкодевочке пройти этаж

Пользователь может сказать:

```text
пройди за меня этот этаж
```

Хост должен вызвать:

```text
sts2_start_autoplay
```

Рекомендуемые параметры:

```json
{
  "objective": "пройди за меня этот этаж",
  "stop_condition": "current_floor"
}
```

Во время выполнения задачи события наблюдения — это лишь отчёты о ходе работы и не означают завершения. Только при получении события завершения полуавтоматической задачи следует сообщить пользователю, что этаж пройден.

### Подсказки в процессе

Во время автоигры пользователь или кошкодевочка могут отправить подсказку:

```text
сначала защищайся, не получай слишком много урона
```

Следует вызвать:

```text
sts2_send_neko_guidance
```

Рекомендуемые параметры:

```json
{
  "content": "сначала защищайся, не получай слишком много урона",
  "type": "soft_guidance"
}
```

Подсказка будет учтена в следующем раунде LLM-решений. Режим `full-program` не зависит от модели, поэтому влияние мягких подсказок ограничено.

## События, отправляемые во фронтенд

Плагин отправляет следующие категории событий через канал сообщений хоста. За исключением старта/завершения задачи, ошибок и анонсов одиночных карт, обычные наблюдения стараются использовать короткий текст и сокращённые metadata, чтобы уменьшить расход токенов пользователя.

| Тип события | Описание |
| --- | --- |
| `action` | Обычное наблюдение действия автоигры, контролируется вероятностью. |
| `error` | Ошибка автоигры, отправляется принудительно. |
| `neko_report` | Полный отчёт наблюдения кошкодевочки, включая текущее состояние, руку, врагов, тактическое резюме и обоснование модели. |
| `neko_card_task_planned` | Задача кошкодевочки по одной карте планирует сыграть определённую карту. |
| `neko_card_task_completed` | Задача кошкодевочки по одной карте выполнена. |
| `neko_card_task_failed` | Задача кошкодевочки по одной карте не может быть выполнена. |
| `semi_auto_task_started` | Полуавтоматическая задача запущена. |
| `semi_auto_task_completed` | Полуавтоматическая задача завершена. |
| `neko_autonomous_action` | Система автономно поставила на паузу, замедлила или возобновила. |

Примечание: `neko_report` — это наблюдение процесса, а не уведомление о завершении задачи. Фронтенд или диалоговая логика не должны описывать одиночное действие, игру карты, завершение хода или обновление состояния как «задача завершена», «босс побеждён», «бой окончен» или «забег пройден». Если кошкодевочка хочет повлиять на следующий раунд решений, следует вызвать `sts2_send_neko_guidance`; если хочет жёстко контролировать процесс, следует вызвать точки входа паузы, возобновления или остановки.

## Частые проверки

### При вызове входов плагина возникает ошибка подключения

Сначала проверьте:

- Запущена ли уже игра.
- Правильно ли Mod `STS2 AI Agent` помещён в `mods/` игры.
- Доступен ли `http://127.0.0.1:8080/health`.
- Правильно ли указан `base_url` в `plugin.toml`.

### `http://127.0.0.1:8080/health` не открывается

Проверьте по порядку:

1. Действительно ли игра уже запущена.
2. Скопированы ли `STS2AIAgent.dll`, `STS2AIAgent.pck` и `mod_id.json` в каталог `mods/` игры.
3. Не были ли файлы переименованы системой, продублированы или помещены не в тот каталог.
4. Работаете ли вы именно в каталоге игры Steam, а не в каталоге исходного репозитория.
5. Не блокирует ли локальный порт брандмауэр или защитное ПО.

### Автоигра работает, но фронтенд не получает сообщения

Проверьте:

- Установлен ли `llm_frontend_output_enabled` в `true`.
- Не слишком ли низок `llm_frontend_output_probability`.
- Установлен ли `neko_reporting_enabled` в `true`.
- Во время интеграционной отладки можно сначала поставить `llm_frontend_output_probability` в `1`.
- Получает ли фронтенд хоста push-сообщения плагина на самом деле.

### Подсказки в середине задачи не дают заметного эффекта

Проверьте:

- Текущий ли режим `half-program` или `full-model`.
- Вернул ли `sts2_send_neko_guidance` значение `ok`.
- Достаточно ли конкретно сформулирована подсказка, например «сначала защита», «бей первым врага с самым низким HP», «сохрани зелье».
- Могут ли текущие допустимые действия реально удовлетворить подсказке.

### Полуавтоматическая задача слишком долго не завершается

Проверьте `stop_condition`:

- Если это `manual` / `none`, задача не завершится автоматически, нужно вызвать `sts2_stop_autoplay`.
- Если это `current_combat`, задача завершится после входа в бой в ходе задачи и последующего выхода из него.
- Если это `current_floor`, обычно завершение происходит после прохождения текущего этажа или перехода на следующий.

Можно вызвать `sts2_get_status`, чтобы посмотреть `autoplay.task`.

### Застревает в событиях, попапах или переходных состояниях

Текущая версия уже обрабатывает события, попапы и переходные состояния. Приоритетные действия включают:

- `confirm_modal`
- `dismiss_modal`
- `choose_event_option`
- `proceed`

Если всё равно зависает, сначала используйте `sts2_get_snapshot`, чтобы проверить текущие `screen` и `available_actions`.

### Автоигра внезапно останавливается или замедляется

Возможно, сработали защитные механизмы:

- Пауза при падении доли HP ниже `neko_auto_low_hp_threshold`.
- Замедление в Boss-боях или опасных атаках.
- Если `neko_auto_resume_after_low_hp` равно `true`, после восстановления HP до `neko_auto_safe_hp_threshold` возможно автоматическое возобновление.

Можно вызвать `sts2_get_status`, чтобы проверить состояние, или использовать `sts2_resume_autoplay` / `sts2_stop_autoplay` для управления.
